ETAPE 1 – SPECIFICATIONS DE GENERATION DU REPO CLOUDSTREAM (16 PLUGINS)
NE PAS COMMENCER LE WEBPANEL A CETTE ETAPE.

============================================================
1. OBJECTIF GENERAL
============================================================

Créer un repo CloudStream propre et minimal contenant 16 plugins (un plugin par site).
Chaque plugin doit être dans son propre dossier et lire ses paramètres (baseUrl, endpoints, selectors, headers…) depuis un fichier JSON de configuration externe.

IMPORTANT :
Aucune URL ne doit être codée en dur dans les plugins.
Le code doit être pensé pour être piloté plus tard par une API WebPanel (étape 2).

SOURCE OFFICIELLE :
Je vais fournir l’URL du repo officiel CloudStream (source d’extensions).  
Tu devras générer un repo 100% compatible CloudStream prêt à être utilisé comme source.  
Dans ce repo, tu ne pousses QUE les fichiers .cs3 + plugins.json (et éventuellement config/sites.json).

============================================================
3. STRUCTURE OBLIGATOIRE DU REPO CLOUDSTREAM
============================================================

Le repo GitHub final utilisé par CloudStream doit contenir UNIQUEMENT :

/plugins.json
/SiteName/SiteName.cs3
/config/sites.json (configuration temporaire modifiable)

Exemple :

/plugins.json
/config/sites.json
/AnimeSamaGF/AnimeSamaGF.cs3
/FrenchStreamGF/FrenchStreamGF.cs3
/WowFilmsGF/WowFilmsGF.cs3
...

IMPORTANT :  
NE PAS pousser dans ce repo :
- le WebPanel
- le backend
- les scripts
- les sources Kotlin
- tout build ou répertoire technique
- node_modules, vendor, bin, build, etc.

Ce repo doit rester propre et minimal.

============================================================
4. DOSSIER PAR SITE
============================================================

Chaque site a son propre dossier :

/SiteName/
/SiteName/SiteName.cs3

Aucun fichier ne doit être partagé entre sites.

Les .cs3 peuvent embarquer leurs helpers internes si besoin.

============================================================
5. FONCTIONNALITES OBLIGATOIRES DE CHAQUE PLUGIN
============================================================

Chaque plugin doit gérer :

- Films
- Séries
- Animés (si disponibles)
- Recherche
- Catégories (si disponibles)
- Liste propre des saisons et épisodes
- Extraction multi-lecteurs (tous les players disponibles)
- Gestion VF / VOSTFR / VO si disponible
- Récupération des qualités (si possible)
- Headers / cookies / user-agent personnalisés
- Endpoints AJAX / API internes
- Code clair, propre, commenté, maintenable

============================================================
6. CONFIGURATION EXTERNE OBLIGATOIRE (PAS D’URL EN DUR)
============================================================

Les plugins lisent leur configuration dans :

/config/sites.json

Exemple :

{
  "FrenchStreamGF": {
    "baseUrl": "https://french-stream.pink/",
    "searchPath": "/recherche",
    "ajaxEndpoint": "/wp-admin/admin-ajax.php"
  },
  "AnimeSamaGF": {
    "baseUrl": "https://anime-sama.eu/"
  }
}

Dans chaque plugin :

val cfg = getSiteConfig("FrenchStreamGF")
val baseUrl = cfg.baseUrl

Le code doit être conçu pour que ce fichier JSON soit remplacé plus tard par une API WebPanel (étape 2).

============================================================
7. GENERATION DU FICHIER plugins.json
============================================================

Tu dois générer un plugins.json contenant les 16 plugins.

Pour chaque plugin :

{
  "name": "FrenchStreamGF",
  "pluginFile": "FrenchStreamGF/FrenchStreamGF.cs3",
  "version": 1,
  "language": "KOTLIN",
  "tvTypes": ["Movie", "TvSeries", "Anime"]
}

============================================================

============================================================

Le code doit être :

- propre
- structuré
- commenté
- stable
- lisible
- réutilisable
- prêt pour la future API WebPanel
